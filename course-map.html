Claude Certified Architect
Knowledge Track

The structured-knowledge layer for the CCA exam: five modules, one per exam domain, covering all 29 task statements. Each module teaches the knowledge the exam tests and points you to the hands-on capstone lab where you build it for real. Work the modules, build the capstones, then drill the practice questions.

⚑ This is a recommended preparatory track, not a certification course. Completing it does not grant a certificate and is not the CCA exam. The exam is booked separately through Anthropic and has its own prerequisites. This track is preparation — pair the knowledge modules with the capstone labs for the strongest footing.

How the track works

Two layers, one path. The knowledge modules below are the structured theory — every task statement, its anti-patterns, and the exam-lens traps. The capstone labs are where you build real systems that exercise those ideas. Each module links to its labs, and each module carries its own check-your-understanding questions — so building the capstones is the best preparation for both.

Suggested path: read a module, build at least one of its linked capstones, then move on. Work the practice questions once the relevant modules are done. The modules are ordered to build on each other, but you can jump to a domain you want to shore up.
The five modules

One per exam domain

All 29 task statements, mapped to five modules that mirror the exam’s own domain structure. Section counts reflect each domain’s weight — Agentic Architecture is the largest.

Module 1 · Domain 1

Agentic Architecture & Orchestration

The agentic loop, coordinator–subagent orchestration, context passing, workflow enforcement, SDK hooks, task decomposition, and session state. The foundation the whole exam rests on.

7 sections Tasks 1.1–1.7 stop_reason · hub-and-spoke · hooks
Module 2 · Domain 2

Tool Design & MCP Integration

Tool interfaces the model selects reliably, structured error responses, scoped tool distribution and tool_choice, MCP server integration, and the built-in tools. What makes an agent pick the right tool and recover when one fails.

5 sections Tasks 2.1–2.5 descriptions · isError · MCP scoping
Module 3 · Domain 3

Claude Code Configuration & Workflows

CLAUDE.md hierarchy and modularity, custom slash commands and skills, path-specific rules, plan mode vs direct execution, iterative refinement, and CI/CD integration.

6 sections Tasks 3.1–3.6 CLAUDE.md · slash commands · plan mode
Module 4 · Domain 4

Prompt Engineering & Structured Output

Explicit criteria to cut false positives, few-shot prompting, structured output via tool use and JSON schemas, validation and retry loops, batch processing, and multi-pass review architectures.

6 sections Tasks 4.1–4.6 JSON schema · validation retry · multi-pass
Module 5 · Domain 5

Context Management & Reliability

Preserving context across long interactions, escalation and ambiguity resolution, error propagation across agents, large-codebase context, human review and confidence calibration, and provenance in multi-source synthesis.

6 sections Tasks 5.1–5.6 escalation · error propagation · provenance

Practice questions

Drill what the modules don’t ask

Each module ends with its own check-your-understanding questions — 89 across the five. The practice sets here are extra: they cover ground the modules teach in prose but never ask you to recall.

They lean deliberately toward the things that are easy to read past and hard to retrieve under exam conditions — the exact flag, the exact frontmatter key, the exact command name. Take them after the modules they draw on, and treat a wrong answer as a pointer back to the module section it came from.
